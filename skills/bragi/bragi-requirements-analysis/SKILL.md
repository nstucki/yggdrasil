# Requirements Analysis

## Purpose

Analyze user requests and transform them into clear, structured requirements.

## Instructions

- Identify the user's underlying goal.
- Extract functional and non-functional requirements.
- Identify constraints, assumptions, and dependencies.
- Detect missing information that affects execution.
- Define acceptance criteria when possible.
- Highlight ambiguities that require clarification.

Focus on understanding what needs to be achieved before deciding how it should be implemented.
